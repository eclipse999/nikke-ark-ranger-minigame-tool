# NIKKE Ark Ranger 小遊戲工具重製計畫書

## 1. 專案目標

本專案目標是重製既有的 NIKKE 小遊戲背包最佳化工具，讓它支援最新活動版本的小遊戲規則。新版工具會先以本地可測試版本完成，後續再視需求部署到 Web。

新版工具需保留原版簡潔、現代、易讀的操作介面，並新增以下能力：

- 支援 9x9 背包棋盤。
- 支援手動切換棋盤格子的可用與不可用狀態。
- 支援 `items-plan.md` 中提供的 P01-P15 全部道具。
- 所有道具皆可旋轉，並自動產生 0、90、180、270 度形狀。
- 自動去除重複旋轉形狀。
- 道具數量由使用者在 UI 手動輸入，不寫死。
- 提供繁體中文與英文介面。
- 先不做截圖辨識，只做手動輸入版。

## 2. 參考介面與使用體驗

新版 UI 參考原版工具的三欄式配置：

- 左側：9x9 背包棋盤。
- 中間：道具清單、形狀預覽與數量輸入。
- 右側：最佳化結果、利用率、解法切換。

視覺風格沿用原版的輕量設計：

- 淺灰背景。
- 白色內容區塊。
- 藍色主要操作按鈕。
- 細格線棋盤。
- 系統字體。
- 簡潔的道具形狀預覽與數字輸入欄位。

載入頁面與「重置背包狀態」時，預設棋盤沿用原版配置：

```text
xxxxxxxxx
xxxxxxxxx
xxx...xxx
xxx...xxx
xxx...xxx
xxx...xxx
xxx...xxx
xxxxxxxxx
xxxxxxxxx
```

其中 `x` 代表不可用格，`.` 代表可用格。介面也會保留「全部可用」按鈕，方便使用者快速切換成完整 9x9 棋盤。

## 3. 技術方案

本地版本採用以下技術：

- Vite
- React
- TypeScript

此方案適合建立可維護的前端工具，也方便之後部署到 Netlify 或其他靜態網站平台。

主要模組規劃如下：

- `items.ts`：定義 P01-P15 道具資料，並產生旋轉形狀。
- 求解器模組：根據棋盤狀態與道具數量計算最佳擺放方案。
- UI 狀態模組：管理棋盤、道具數量、語言、目前選取解法。
- 語系資料：提供繁體中文與英文文案。
- React 元件：棋盤、道具列表、結果棋盤、控制列、語言切換。

## 4. 道具資料規則

道具資料只使用 `items-plan.md` 中提供的 P01-P15，不額外假設任何未提供道具。

每個道具以 `X` 和 `.` 表示：

- `X`：佔用格。
- `.`：空格。

程式會自動完成以下處理：

- 解析原始形狀。
- 正規化形狀座標。
- 產生 0、90、180、270 度旋轉版本。
- 去除重複旋轉結果。
- 計算每個形狀的寬、高、面積與佔用座標。

建議資料型別：

```ts
type Cell = {
  row: number;
  col: number;
};

type Shape = {
  cells: Cell[];
  width: number;
  height: number;
  area: number;
  rotation: 0 | 90 | 180 | 270;
};

type ItemDefinition = {
  id: string;
  baseShape: string[];
  rotations: Shape[];
};
```

## 5. 求解器設計

求解器輸入包含：

- 9x9 棋盤可用狀態。
- 每種道具的使用者輸入數量。
- 最大候選解數量。
- 搜尋時間上限。

求解器輸出包含：

- 最佳填入格數。
- 利用率。
- 候選解法清單。
- 是否已完整證明最佳解。
- 若未完整搜尋，需標示停止原因。

求解策略：

- 使用回溯搜尋。
- 優先嘗試面積較大、限制較高的道具。
- 對每個道具嘗試所有去重後旋轉形狀。
- 檢查形狀是否能放入目前棋盤空位。
- 使用剩餘可用格與剩餘道具面積做剪枝。
- 保留少量最佳或接近最佳候選解供 UI 切換。
- 若搜尋超過時間上限，回傳目前找到的最佳方案，並標示為「時間內最佳」。

此設計能在新版「所有道具可旋轉」的規則下控制計算量，同時避免使用者輸入較大數量時整個頁面卡死。

## 6. 使用者功能

本地測試版需提供以下功能：

- 點擊棋盤格切換可用與不可用。
- 重置棋盤狀態。
- 將棋盤設為全部可用。
- 為 P01-P15 分別輸入數量。
- 清空全部道具數量。
- 執行最佳化計算。
- 顯示最佳利用率。
- 顯示最佳擺放結果。
- 切換候選解法。
- 顯示求解狀態，例如「已證明最佳」或「時間內最佳」。
- 切換繁體中文與英文介面。

## 7. 不在本階段範圍內

以下功能不列入第一版本地工具：

- 截圖辨識。
- 自動讀取遊戲畫面。
- 使用者帳號或資料同步。
- 雲端儲存。
- Web 部署。
- 新增 `items-plan.md` 未提供的道具。

## 8. 測試計畫

### 單元測試

- 測試 P01-P15 是否正確解析。
- 測試旋轉產生是否正確。
- 測試 P07、P11 等對稱形狀會去除重複旋轉。
- 測試非對稱形狀會保留多個有效旋轉。
- 測試棋盤放置合法性判斷。
- 測試無道具、單格道具、無法放入道具等求解情境。

### UI 測試

- 點擊棋盤格後狀態正確切換。
- 「重置背包狀態」回到原版預設棋盤。
- 「全部可用」產生完整 9x9 可用棋盤。
- 道具數量輸入與清空功能正常。
- 計算結果能正確更新利用率與結果棋盤。
- 候選解法切換正常。
- 語言切換後文案正確更新。

### 視覺與響應式測試

- 桌面寬度下維持三欄式配置。
- 窄螢幕下可改為單欄或分段堆疊。
- 棋盤格、道具預覽、按鈕文字不重疊。
- 結果棋盤與輸入區不因內容變動產生明顯跳動。

## 9. 驗收標準

第一版本完成時，需符合以下條件：

- 本地 dev server 可正常啟動。
- 使用者可在瀏覽器中操作完整流程。
- P01-P15 道具皆出現在 UI 中。
- 道具旋轉規則正確生效。
- 求解結果能顯示最佳或時間內最佳方案。
- 繁體中文與英文介面皆可使用。
- 不包含截圖辨識與部署相關功能。
